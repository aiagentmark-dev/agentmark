// mechanism

four proofs in every commit

agentmark combines cryptographic primitives into a chain of evidence. Each proof is independently verifiable.

01 / output_hash
no-edit proof
sha256(raw_response) == sha256(committed_code)

Proves the code committed matches the LLM's raw HTTP response byte-for-byte. Any edit — even a single space — breaks the hash.
02 / challenge_token
llm-saw-this proof
A unique token is embedded in the task. The LLM must echo it in its response. The echo is covered by output_hash.

Proves the LLM processed this specific task — not a replay or fabrication.
03 / request_id
real-api-call signal
Every cloud provider returns a unique request_id header per API call. Captured and included in the manifest.

Proves a real API call happened. Cannot be fabricated without making the actual call.
04 / Ed25519 signature
pipeline identity proof
Every registered pipeline holds an Ed25519 private key. The manifest is signed and the signature verified against the public key in the agentmark registry.

Proves a registered pipeline made this commit.
// honest limits

what agentmark cannot prove

We document failure modes explicitly. There are no false guarantees.

F1
The verbatim human copy attack. A human who runs the complete agentmark-compliant pipeline (real API call, verbatim commit, valid manifest) is indistinguishable from an agent. This is by design — they have built an agent.
F2
LLM proxy / gateway breaks output_hash. Agents must call provider APIs directly. LiteLLM, Bedrock, and Azure proxies transform responses and break hash verification.
F3
Local models have no request_id. Ollama, llama.cpp, and self-hosted models cannot provide provider-issued request IDs. Tier 2 trust applies — output_hash and challenge_token only.

Full list of 8 failure modes in SPEC.md §4.

// quickstart

integrate in minutes

Three integration points. No architecture changes required.

install 
pip install agentmark
wrap your llm call 
import agentmark

# wrap your existing LLM call
result = agentmark.call(
    provider="anthropic",
    model="claude-sonnet-4-20250514",
    prompt=prompt,
    challenge_token=challenge,  # issued by agentmark GitHub App
)

# sign and build manifest
manifest = agentmark.sign(
    result=result,
    pipeline_key="my-pipeline-v1",
    private_key_bytes=key,
)

# manifest goes in commit message — done
add ci gate (.github/workflows/agentmark-verify.yml) 
- name: Verify pipeline provenance
  uses: aiagentmark-dev/agentmark-action@v1
  with:
    strict: true
